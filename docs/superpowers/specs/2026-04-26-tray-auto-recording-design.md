# タスクトレイ常駐 + 自動録音機能 設計仕様

## 概要

議事録AIアプリをシステムトレイに常���させ、Zoom / Teams / Google Meet のウィンドウを自動検知して録音を開始・停止する機能を追加する。ユーザーはボタンを押す必要なく、会議アプリを開くだけで自動的に議事録が生成される。

## 動作フロー

```
PC起動 → トレイに常駐（アイコン表示）
    ↓
5秒ごとにウィンドウタイトルをスキャン
    ↓
"Meet" / "Zoom" / "Teams" を検知
    ↓
自動でオンラインモード録音開始（トレイアイコンが赤に変化）
    ↓
ウィンドウが消える → 5分カウントダウン
    ├── 5分以内に復帰 → カウントダウン解除、録音継続
    └── 5分経過 → 録音停止 → 議事録生成パイプライン実行
    ↓
完了 → デスクトップ通知「議事録が完成しました」
    ↓
待機状態に戻る（次の会議を監視）
```

## ウィンドウ検知ルール

`win32gui.EnumWindows()` で全ウィンドウのタイトルを取得し、以下のパターンにマッチするか判定する（大文字小文字無視）:

| アプリ | 検知パターン | カバー範囲 |
|--------|-------------|-----------|
| Zoom | `"zoom meeting"`, `"zoom webinar"` | デスクトップ版 + ブラウザ版 |
| Teams | `"microsoft teams"` | デスクトップ版 + ブラウザ版 |
| Google Meet | `"meet - "` | ブラウザ版（Chrome, Edge等） |

ブラウザのタブタイトルにはアプリ名が含まれるため、デスクトップ版・ブラウザ版の両方を同一ロジックで検知できる。

## 録音モード

- 自動検知時: 常に**オンラインモード**（画面キャプチャ + マイク音声）
- 対面会議: トレイメニューから**手動で開始**

## 猶予時間（Grace Period）

ウィンドウが消えた後、5分間の猶予時間を設ける。

- 猶予時間中に会議ウィンドウが再検知された場合 → カウントダウン解除、録音継続
- 猶予時間経過後も会議ウィンドウがない場合 → 録音停止、議事録生成パイプライン実行

目的: 通信トラブルによる一時的な切断や、ブラウザタブの誤閉じからの��帰に対応する。

## 新規モジュール

### `tray/app.py` — システムトレイアプリケーション

pystray を使用してシステムトレイにアイコンを常駐させる。

**トレイアイコン状態:**
- 待機中: 通常アイコン（グレー系）
- 録音中: 赤いアイコン
- 処理中（議事��生成中）: 黄色アイコン

**トレイメニュー:**
- 「対面モードで録音開始」— 手動で対面モード録音を開始
- 「録音停止」— 現在の録音を手動停止し議事録生成パイプラインを実行
- 「設定」— セットアップ画面（ui/setup.py）を開く
- 「終了」— アプリケーションを終了

### `tray/monitor.py` — ウィンドウタイトル監視

5秒間隔で全ウィンドウのタイトルをスキャンし、会議アプリの存在を判定するモジュール。

**責務:**
- `win32gui.EnumWindows()` による全ウィンドウタイトル取得
- 検知パターンとのマッチング
- 会議ウィンドウの検知状態（検知中 / 非検知）を返す

### `tray/icons.py` — アイコン生成

Pillow を使用してトレイアイコン画像をプログラム的に生成するユーティリティ。外部画像ファイルへの依存を避ける。

## 既存モジュールの変更

### `main.py`

エントリーポイントを変更:
- `python main.py` → トレイ常駐モードで起動（デフォルト）
- `python main.py --gui` → 従来のTkinter GUI で起動
- 初回起動時（`setup_complete` が未設定）はセットアップウィザードを表示してからトレイモードへ

### `requirements.txt`

以下のパッケージを追加:
- `pystray>=0.19.0` — システムトレイ
- `Pillow>=10.0.0` — アイコン画像生成
- `pywin32>=306` — ウィンドウタイトル取得（`win32gui`）

## 状態遷移

```
[IDLE] ---(会議ウィンドウ検知)---> [RECORDING]
[RECORDING] ---(会議ウィンドウ消失)---> [GRACE_PERIOD]
[GRACE_PERIOD] ---(5分以内に復帰)---> [RECORDING]
[GRACE_PERIOD] ---(5分経過)---> [PROCESSING]
[PROCESSING] ---(パイプライン完了)---> [IDLE]
[RECORDING] ---(手動停止)---> [PROCESSING]
```

状態は `tray/app.py` の `TrayApp` クラスが管理する。

## テスト方針

- `tray/monitor.py`: `win32gui.EnumWindows` をモックしてウィンドウタイトル検知ロジックをテスト
- `tray/app.py`: 状態遷移ロジック（IDLE → RECORDING → GRACE_PERIOD ��� PROCESSING → IDLE）をテスト
- `tray/icons.py`: アイコン画像生成が正しいサイズ・カラーで生成されることをテスト
