# 議事録生成AI - 設計ドキュメント

## 1. プロジェクト概要

対面会議・オンライン会議の音声/映像を自動で録音・録画し、AIによる文字起こしと議事録生成を行うデスクトップアプリケーション。  
ワンボタンで起動でき、会議終了後に議事録が自動生成されてクラウドストレージに保存される。

### コンセプト
- **ボタン1つで起動** — 複雑な設定不要
- **全情報を漏れなく記録** — 要約ではなく、全発言を構造化して残す
- **完全無料** — すべてのコンポーネントを無料枠で構成

---

## 2. システム構成

### 技術スタック

| コンポーネント | 技術 | 費用 |
|--------------|------|------|
| デスクトップUI | Python + Tkinter | 無料 |
| 録音/録画 | FFmpeg（バックグラウンド実行） | 無料 |
| クラウドストレージ | Google Drive API / Microsoft OneDrive (Graph API) | 無料 |
| 文字起こし + 議事録生成 | Gemini API（無料枠） | 無料 |

### Gemini API 無料枠

- モデル: Gemini 2.5 Flash（音声入力対応、100万トークンコンテキスト）
- レート制限: 15リクエスト/分、1,500リクエスト/日
- 音声を直接入力できるため、別途ASRエンジン不要

### クラウドストレージ比較

| | Google Drive | OneDrive |
|--|-------------|----------|
| 無料容量 | 15GB | 5GB |
| API | Google Drive API | Microsoft Graph API |
| 認証 | Googleアカウント（OAuth 2.0） | Microsoftアカウント（OAuth 2.0） |

ユーザーが初回セットアップ時にどちらを使用するか選択する。

## 4. 処理フロー

### 全体パイプライン

```
ボタン押す（対面 or オンライン）
  ↓
FFmpegがバックグラウンドで録音/録画開始
  ├── 対面モード: マイク音声 → .wav
  └── オンラインモード: 画面+音声 → .mp4 + .wav
  ↓
30分ごとにファイルを自動分割（長時間会議対応）
  ↓
停止ボタン押す
  ↓
各セグメントをクラウドストレージに自動アップロード
  ├── Google Drive API
  └── OneDrive (Microsoft Graph API)
  ↓
Gemini APIに音声ファイルを送信
  ├── 各チャンクを並列処理（文字起こし＋話者識別）
  └── タイムスタンプをオフセット補正して全チャンクを結合
  ↓
結合した文字起こしからGemini APIで議事録を生成
  ↓
完成した議事録をクラウドストレージに保存
  ↓
デスクトップ通知（「議事録が完成しました」）
```

### 長時間会議への対応（60分超）

Gemini APIの音声入力制限に対応するため、録音中にFFmpegで自動的にセグメント分割する。

```bash
# 録音しながら30分ごとにファイルを分割
ffmpeg -f dshow -i audio="マイク" \
  -f segment -segment_time 1800 \
  -reset_timestamps 1 \
  meeting_%03d.wav
```

- 各セグメント（30分ごと）を個別にGemini APIへ送信
- タイムスタンプにオフセットを加算して時系列を維持
- 隣接チャンクをオーバーラップさせてトピック切れ目を補完

## 5. 議事録の出力形式

### 方針
- **要約ではなく全量構造化** — 全ての発言を漏れなく記録
- フィラー（「えーと」「あのー」等）は除去
- 口語を読みやすい文体に整文化（内容は変えない）
- トピックごとにセクション分け + タイムスタンプ付与
- 話者ラベルを整理（Speaker1 → 実名）

### 出力例

```markdown
# 定例会議 2026-04-26

## 参加者
山本、田中、佐藤

---

## 1. プロジェクト進捗報告（00:00〜15:32）

**山本:** 先週のスプリントでAPI実装が完了した。
テストカバレッジは85%まで上がっている。

**田中:** フロント側の結合テストはまだ未着手。
来週水曜までに完了予定。

**佐藤:** デザインレビューで戻しが2件あった。
ボタン配置とフォントサイズの修正が必要。

## 2. 予算について（15:33〜28:10）

**山本:** 今四半期の予算消化率は62%。
残り予算でインフラ増強を検討したい。

**田中:** AWSのリザーブドインスタンスに切り替えれば
月額15%のコスト削減が見込める。具体的な試算を来週出す。

**佐藤:** デザインツールのライセンスが来月更新。
Figmaのビジネスプランは年間で1人あたり$45。チーム全体で5ライセンス必要。

...（全発言が続く）

---

## アクションアイテム
- [ ] 田中: フロント結合テスト完了（〜来週水曜）
- [ ] 田中: AWSコスト試算を提出（〜来週）
- [ ] 佐藤: ボタン配置・フォントサイズの修正
- [ ] 佐藤: Figmaライセンス更新の稟議書作成
```

## 6. クラウドストレージのフォルダ構成

```
議事録AI/
  └── 2026-04-26_定例会議/
       ├── recording_000.wav      ← 録音データ（セグメント1）
       ├── recording_001.wav      ← 録音データ（セグメント2）
       ├── recording.mp4          ← 録画（オンラインモード時のみ）
       ├── transcript.txt         ← 文字起こし全文
       └── minutes.md             ← 議事録（構造化済み）
```

## 7. ユーザー初回セットアップ

1. アプリをインストール（Python環境 + FFmpeg）
2. クラウドストレージを選択（Google Drive or OneDrive）
3. OAuth認証でクラウドストレージと連携
4. Gemini APIキーを設定（Google AI Studioから無料で取得）
5. マイクデバイスを選択
